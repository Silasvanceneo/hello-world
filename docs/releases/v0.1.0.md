# hello-world v0.1.0

This release packages the first complete `hello-world` local-first AI client roadmap across Web, Windows Desktop, and Android Mobile.

## Highlights

- Native provider runtime for OpenAI-compatible endpoints, OpenAI Responses, Anthropic Messages, Gemini, Azure OpenAI, DashScope, Ollama, and relay gateways.
- Mature local-first RAG with ingestion, chunking, deterministic local embeddings, hybrid lexical/vector retrieval, citations, reranking hooks, and eval reporting.
- Web search provider normalization for Brave, Tavily, Bing, SearXNG, and custom endpoints with grounded answer citation validation.
- HTTP MCP as the shared safe tool subset across Web, Desktop, and Mobile.
- Desktop-only stdio MCP/plugin control plane behind policy and confirmation gates.
- Desktop-only sandboxed JavaScript/Python code execution through a controlled Tauri command, with no arbitrary terminal or shell endpoint.
- Bilingual English/Chinese UI, dedicated Settings workspace, advanced capability configuration, app icon integration, tray icon, and local build artifacts.

## Security Notes

- Provider and search API keys are runtime-only and are not persisted in local state, backup archives, screenshots, or release metadata.
- Terminal, shell, cmd, PowerShell, arbitrary process spawning, network proxy, and broad filesystem access remain blocked by default.
- Desktop code execution supports only controlled JavaScript/Python snippets with timeout, output, environment, and confirmation limits.
- Web and Mobile do not expose stdio MCP, Desktop proxy, sandboxed code execution, or terminal access.

## Known Boundaries

- Web provider calls are still subject to browser CORS. Use a compatible relay or the Desktop surface when direct provider CORS blocks requests.
- Mobile currently ships the Android path; iOS directories are scaffolded but not release-built in this checkpoint.
- The plugin manager is a Desktop control plane and does not provide a public marketplace yet.
